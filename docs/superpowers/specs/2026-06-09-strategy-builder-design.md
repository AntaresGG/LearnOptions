# Strategy Builder 页面设计（精简 spec）

日期：2026-06-09 · 文件：`strategy-builder.html`（找回旧名，独立单屏 builder）

## 目标
基于 `strategy-system.html` 的「策略实验室 · 矩阵选型与图鉴」体系，构建一个独立的、**单屏**期权策略 builder：选型信息、控制要素、损益图在一屏内全部可见。区别于教学模块，采用满宽布局与独立配色。

## 已定决策
- 构建本质：**预设选型 + 完整行内编辑/增删腿**（融合 06-payoff 的多腿搭建器能力）。
- 布局：**左控制栏 / 右图表**，左 ≈ 400px，右弹性大图。
- 适配：**严格桌面专用**，不做手机响应式。
- 留白：**满宽**，最小化左右留白，最大化利用屏宽（不同于其他页面的居中 max-width）。
- 配色：**独立「Quant Desk」风格** —— 冷调炭黑底 + 紫罗兰/靛色强调（`--accent #a78bfa`）+ Inter 无衬线 + JetBrains Mono 等宽数字。语义色保留：红涨 `--up` / 绿跌 `--down` / Call 蓝 / Put 橙。
- 权利金：统一按 Black-Scholes 实时估算（S₀=100, r=2%），随 σ / T 滑块重算。

## 页面族归属与复用
属核心族（用 `OA`）。按项目「复制粘贴、不共享」规则，从 `strategy-system.html` 内联拷贝：
- `OA` 库（`window.OA`）完整原样复制（保持 `window.OA =` 家族一致）。
- 数据：`DEF`（19 个策略，剔除 `calendar`）、`CAT` 四大家族、`MX` 选型矩阵（`震荡|低` 推荐改为 `butterfly`）。
- 辅助：`P/opt/stk/legVal/legPL`（去掉 `far/tRem` 远月分支，因已剔除 calendar）。

## 交互
1. 点矩阵选型 → `MX` 推荐 → 用 `DEF[k].legs()` 填充腿编辑器并显示推荐/备选。
2. 点图鉴芯片 → 填充对应预设策略。
3. 腿编辑器：每行可改 类型(Call/Put/Stock)、方向(Long/Short)、K、数量；可加腿/删腿。任意编辑后标题切换为「自定义组合 Custom」。
4. σ / T 滑块 → 所有腿权利金按 `P(kind,K)` 重算 → 曲线/指标刷新。
5. 腿状态仅存 `{kind,dir,K,qty}`，`entry` 在 render 时按当前 σ/T 现算，保证一致。

## 指标计算
沿用父页宽区间扫描（S∈[0,300] step .25）求 max/min/盈亏平衡点；净成本 = Σ −dir·qty·entry。
最大盈利/亏损的「∞/无限」判定改用**右端斜率检测**（右端仍上行→最大盈利 ∞；右端仍下行→最大亏损 −∞），左端以 S=0 为天然下界，比依赖 DEF.mp 更适配自由编辑后的组合。

## 不做（YAGNI）
手机响应式、策略保存/分享、calendar 远月、改动其它页面、index 导航链接。

## 实现路径
